# Cooking Dataset Training (EgoX QLoRA on 24 GB Blackwell)

Reference for the small-scale EgoX reproduction trained on the **cooking** subset of Ego-Exo4D,
on a single 24 GB RTX PRO 4000 Blackwell GPU. Last updated 2026-06-23.

---

## 1. Goal

Train EgoX (exo→ego video generation) on cooking clips using **QLoRA** (NF4 4-bit Wan2.1-I2V-14B
base + trainable LoRA), at a **reduced spatial resolution** so it fits 24 GB. Pre-encode the
dataset once (latents/embeds/GGA cached, encoders unloaded) before the training loop.

## 2. Training resolution — LOCKED `49×256×704`

- **Frames are hard-locked at 49** (`wan_dataset.py:128` assert + `ego_extrinsic[::4]` hardwiring +
  49 depth `.npy` + meta `ego_extrinsics` is 49-long). Only **spatial H×W** is reducible.
- Paper trains at `49×448×1232` (won't fit 24 GB: GGA cos-sim N² buffer alone = 3.14 GB at N=28028).
- Chosen `49×256×704` (N≈9,152, GGA buffer ~0.33 GB). Validated: Phase A peaked **22.8 GB / 24 GB**.
- Resolution is a **load-time** setting (`--train_resolution`); the loader downsamples on decode and
  caches reduced latents. **Originals are never resized.** Token math: N = 13·(H/16)·(W/16).

## 6. Stage-1 generation pipeline (per take → its clips)

```
cam04.mp4 (796x448 full take)
  └─ vipe infer -p lyra            -> depth.zip + intrinsics.npz + point cloud/poses   [#4,#5]
       |- convert_depth_zip_to_npy -> depth_maps/<take>/*.npy                          [#4]
       |- render_vipe_pointcloud   -> ego_Prior.mp4 (fisheye)                          [#3]
       |- trim cam04 + aria        -> exo.mp4 + ego_GT.mp4 (49-frame windows)          [#1,#2]
  -> assemble training-schema meta.json
  -> (training) precompute downsamples to 49x256x704 + caches latents
```
**Order:** ViPE/Stage-1 at the native 796×448 FIRST; resolution reduction to 256×704 happens LAST,
automatically, in the trainer's precompute (never pre-resize the files).
**Note:** `vipe infer` alone produces only depth + intrinsics; ego_prior (render) + clip trims are
**separate steps** that consume ViPE's output.

## 7. `online_calibration.jsonl` decision — SETTLED

- **Not generated** — it's an Ego-Exo4D dataset file (`take_trajectory` part), recorded Aria calib.
- **Not needed / not downloaded** — the renderer has **built-in generic Aria distortion coeffs**
  (= the set `infer.py:139` uses for inference) when the file is absent. The 54 GB `take_trajectory`
  download was **cancelled**.
- The authors' faithful `data_preprocess/` pipeline uses per-take calib, but the benefit is marginal
  (Aria RGB cams are near-identical). **Use the built-in generic coeffs.**
- ⚠️ Open render-config bug: the earlier fisheye render gave a circular-180° projection (from the
  `--fish_eye_rendering` flag/FOV), not a calibration problem — needs a render-config fix so it
  produces the frame-filling Aria fisheye that aligns with the ego_GT.
